The Controls.

The first set of controls determine which algorithm is going to be run. These are extensible by the user.

Type
The type of FROG being used (e.g. FROG, XFROG, etc.).
Nonlinearity
The particular nonlinearity used (e.g. SHG, PG, THG, SFG, OPA, etc.).
Domain
The domain of the algorithm (e.g. Time, Frequency)
Algorithm
The underlying algorithm of the retrieval (e.g. Z minimize (projection), XFROG, etc.).

Buttons

Run
Starts the algorithm. Disabled when running. Same functionality provided through the Execute menu.
Stop
Stops the algorithm. Disabled when not running. Same functionality provided through the Execute menu.
Extra Information
If the algorithm needs extra information to run, this button is enabled. For example, XFROG needs the gate pulse in order to run. This button is used to bring up a dialog to select the apropriate gate pulse. However, this can be used in any algorithm that needs extra input from the user.
Flip Time
Primarily implemented for SHG FROG. This allows the user to flip the pulse in time (the current, Best G, and Best Z pulses). If the user knows which edge of the pulse is leading, this can fix it. It can also be used to get other nonlinearities out of local minima.
Flatten Phase
Provided to kick the algorithm out of local minima.
